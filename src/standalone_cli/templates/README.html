2. Acronyms

  • AIP: Archival Information Package
  • DIP: Dissemination Information Package
  • SIP: Submission Information Package
  • METS: Metadata Encoding and Transmission Standard
  • PREMIS: Preservation Metadata: Implementation Strategies
  • UUID: Universally Unique Identifier

3. AIP Structure

This AIP is packaged according to the BagIt specification.

  • data/: Contains the payload content.
    • objects/: Original and normalized digital objects.
    • logs/: Processing logs.
    • thumbnails/: Generated thumbnails.
    • METS.{{ sip_uuid }}.xml: The primary metadata file describing the AIP.
  • bagit.txt: BagIt version and encoding.
  • bag-info.txt: Metadata about the bag.
  • manifest-sha512.txt: Checksums of payload files.
  • tagmanifest-md5.txt: Checksums of tag files.

4. Content Information

The objects directory contains the digital objects. If normalization was performed, original files are preserved alongside their normalized versions (or in a structured manner defined by the METS structMap).

5. Preservation Description Information (PDI)

Preservation metadata is recorded in the METS file (METS.{{ sip_uuid }}.xml) using the PREMIS standard. This includes information about:

  • Provenance: Who created the files and how they were processed.
  • Fixity: Checksums to ensure file integrity.
  • File Format: Identification of file types (e.g., via FIDO).
